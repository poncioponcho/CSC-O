# 基于 8× GTX 1080 Ti 的算力增强实验方案

> **当前算力基线**: 8 × GTX 1080 Ti (11GB × 8 = 88GB), 125GB RAM, CUDA 11.4  
> **当前耗时**: ESM-2 (t12_35M) 49s / Counterfactual 222s / 总计 ~6-8min  
> **算力利用率**: ESM-2 仅用了单卡，7 卡空闲；Counterfactual 未用 GPU

---

## 一、算力现状诊断

| 组件 | 当前使用 | 可用余量 | 瓶颈 |
|------|---------|---------|------|
| GPU 显存 | ~2GB (单卡跑 t12) | **86GB 空闲** | 模型太小 |
| GPU 利用率 | ~15% | **85% 空闲** | 未并行 |
| CPU/内存 | 轻度 | 充足 | 非瓶颈 |
| 磁盘 I/O | 轻度 | 充足 | 非瓶颈 |

**核心问题**: 你花了 88GB 显存的硬件，却只用了 2GB 跑一个 35M 参数的小模型。这相当于用 8 辆卡车运一张 A4 纸。

---

## 二、推荐实验方案（按优先级排序）

### 方案 A: ESM-2 模型升级 + 多尺度嵌入融合 ⭐⭐⭐（首推，立即可做）

**核心思路**: 从 t12_35M 升级到 t30_150M / t33_650M，融合多尺度表示。

**为什么值得做**:
- t12_35M 只有 480 维嵌入，表达能力有限
- t30_150M (640 维, 30 层) 和 t33_650M (1280 维, 33 层) 能捕获更复杂的序列模式
- 文献表明，ESM-2 650M 在抗体任务上比 35M 提升 5-15%

**技术实现**:
```python
# 当前
model_name = "esm2_t12_35M_UR50D"  # 480 dim

# 升级方案
models = {
    "t12_35M":  "esm2_t12_35M_UR50D",   # 480 dim,  ~2GB
    "t30_150M": "esm2_t30_150M_UR50D",   # 640 dim,  ~6GB  ← 单卡可跑
    "t33_650M": "esm2_t33_650M_UR50D",   # 1280 dim, ~14GB ← 需2卡模型并行
}
```

**显存规划** (8 × 11GB):
| 模型 | 单卡显存 | 并行策略 | 预计编码时间 |
|------|---------|---------|-------------|
| t12_35M | 2GB | 单卡 | 49s (已验证) |
| t30_150M | 6GB | 单卡/数据并行 | ~3-5min |
| t33_650M | 14GB | **2卡模型并行** | ~8-12min |

**融合策略**:
- 拼接融合: `[t12_emb (480) | t30_emb (640) | t33_emb (1280)]` → 2400 维
- 或加权融合: `α·t12 + β·t30 + γ·t33`
- 下游任务用降维 (PCA→512 维) 或直接用 LightGBM 处理高维

**预期收益**:
- CATE 估计精度提升
- 反事实建议更准确
- 成功率预测 AUC 提升 5-10%

### 方案 B: 因果森林 (Causal Forest) 替代常数 CATE ⭐⭐⭐

**核心思路**: 当前 Double ML 估计的是**常数 CATE**（所有样本效应相同），但现实中不同序列对同一突变的响应不同。因果森林可以估计**异质性 CATE**。

**为什么值得做**:
- 能回答"哪类序列对 Pos0 G→Y 响应最好？"
- 发现亚群特异性效应（如：长序列 vs 短序列的突变效应不同）
- 这是因果推断的前沿方法，论文价值高

**技术实现**:
```python
# 当前: 常数 CATE (所有样本一个 theta)
cate = double_ml_cate(X, T, Y)  # → 标量

# 升级: Causal Forest (每个样本一个 CATE)
from econml.grf import CausalForest
est = CausalForest(n_estimators=500, 
                   criterion='mse', 
                   n_jobs=8)  # ← 用满8核CPU
est.fit(X, T, Y)
cate_individual = est.effect(X)  # → 每个样本的个性化效应
```

**计算成本**:
- 训练 500 棵树的森林: ~5-10 分钟 (CPU 并行)
- 8 核 CPU 可以跑满

**预期收益**:
- 发现"亚群特异性规则"（如：首残基为 G 的序列对 Y 替换响应最强）
- 生成更精确的反事实建议

### 方案 C: 条件序列生成器 + 快速筛选流水线 ⭐⭐⭐（最具应用价值）

**核心思路**: 当前分析是"诊断性"的（分析已有数据），下一步应该是"生成性"的（主动设计新序列）。

**流水线设计**:
```
Step 1: 生成器 (Generator)
    输入: design_strategy.json 的约束条件
    模型: 微调 ProtGPT2 / ESM-2 作为生成器
    输出: 10,000 条符合约束的候选 CDR3 序列

Step 2: 判别器 (Discriminator)  
    输入: 候选序列 → ESM-2 (t33_650M) 嵌入
    模型: LightGBM / Transformer
    输出: RF2 通过概率 + 预测 PAE

Step 3: 筛选 (Filter)
    保留 Top 200 (预测通过率 > 40%)
    
Step 4: 验证 (Validation)
    用 RF2 / AF3 对 Top 200 进行结构预测
    
Step 5: 迭代 (Iteration)
    将验证结果反馈给生成器，微调策略
```

**为什么值得做**:
- 从"分析过去"转向"设计未来"
- 这是抗体工程的实际需求：给出可实验验证的新序列
- 论文档次显著提升（生成 + 验证闭环）

**计算成本**:
- 生成 10,000 条: <1 秒
- ESM-2 编码 (t33_650M, 2卡并行): ~8-12min
- LightGBM 评分: <1 秒
- **总计: ~10-15 分钟生成一轮候选**

**预期收益**:
- 生成符合策略的序列，预测通过率可达 40%+
- 提供可直接送 RF2/AF3 验证的候选列表

### 方案 D: 多任务深度预测网络 (Multi-Task Transformer) ⭐⭐

**核心思路**: 当前每个阶段是独立的（Cox → ATE → CATE → 合成）。可以训练一个统一的深度模型，同时预测多个目标。

**架构**:
```
输入: VH 序列
    ↓
ESM-2 (t33_650M, 可微调)
    ↓
共享 Transformer Encoder
    ↓
├── 任务头1: RF2 通过概率 (Binary Classification)
├── 任务头2: 预测 PAE (Regression)
├── 任务头3: 预测 AF3 rank (Regression)
├── 任务头4: 预测 binding affinity (Regression)
└── 任务头5: 最终候选概率 (Binary Classification)
```

**损失函数**:
```python
loss = λ1·BCE(rf2_pred, rf2_true) + \
       λ2·MSE(pae_pred, pae_true) + \
       λ3·MSE(af3_pred, af3_true) + \
       λ4·MSE(binding_pred, binding_true) + \
       λ5·BCE(candidate_pred, candidate_true)
```

**计算成本**:
- ESM-2 冻结 + 5 个任务头微调: ~30-60 分钟
- 或全量微调 (需更大显存): 可能需要 4 卡数据并行

**预期收益**:
- 一个模型替代当前多阶段流水线
- 端到端预测，推理速度极快
- 可以在线评分新设计的序列

### 方案 E: 图神经网络 (GNN) 结构特征增强 ⭐⭐（需结构数据）

**核心思路**: 当前只用序列特征（ESM-2）。如果补充结构特征（二级结构、接触图、溶剂可及性），可以构建序列-结构联合表示。

**前提条件**:
- 需要抗体结构数据（PDB 或 AlphaFold 预测结构）
- 或用 ESMFold / OmegaFold 批量预测结构

**技术实现**:
```python
# 结构预测 (批量)
from esm import ESMFold  # 或 ColabFold
structure = esm_fold.predict(sequence)  # 每个序列 ~10-30s

# 结构特征提取
from Bio.PDB import DSSP
ss = dssp.get_secondary_structure(structure)  # α/β/coil

# GNN 建模
import dgl
graph = build_contact_graph(structure)  # 残基接触图
emb = gatconv(graph, esm2_emb)  # 图注意力网络
```

**计算成本**:
- ESMFold 预测 10,572 条序列: ~3-5 小时 (8卡并行)
- GNN 训练: ~30-60 分钟

**预期收益**:
- 结构信息能捕获序列无法表达的空间约束
- 对 binding affinity 预测提升显著

## 四、显存管理策略
