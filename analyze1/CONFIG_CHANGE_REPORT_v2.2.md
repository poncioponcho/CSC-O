# CSC-O 配置变更技术报告 v2.1 → v2.2

## 一、变更概述

基于v2.1运行结果的全面分析，发现放宽RF2阈值后新增的1673条"边缘通过"序列在下游AF3阶段通过率为0%，FC数为0，属于"虚假通过"。本报告详细记录v2.2配置调整的每项变更、数据支撑、预期收益和潜在风险。

## 三、每项变更的详细分析

### 3.1 lddt阈值 0.85 → 0.86

**数据支撑**：
- v2.1新增1673条中，lddt [0.85,0.86)区间167条，FC=0, AF3=0, avg_PAE=9.35
- lddt [0.86,0.87)区间540条，FC=0, AF3=0, avg_PAE=8.09
- lddt=0.86剔除295条，FC损失=0，首残基G占70%

**预期收益**：
- 剔除295条低质量序列（avg_PAE=8.26, avg_lddt=0.850）
- RF2通过数从3173→2878，质量提升（avg_PAE 6.27→6.06）
- 减少下游AF3计算资源浪费

**潜在风险**：
- 风险极低：剔除的295条FC=0，无任何有效候选损失
- 可能遗漏极少数lddt=0.85但PAE极低的序列（但数据显示此类序列PAE均值9.35，质量确实差）

**性能影响**：
- RF2通过数减少9.3%，但有效候选数不变
- 下游计算量减少约9%，AF3/Schrodinger阶段加速

### 3.2 PAE阈值 12.0 → 10.0

**数据支撑**：
- v2.1新增1673条中，PAE [10,12)区间300条，FC=0, avg_lddt=0.862
- v2.0原有1235条中，PAE范围[3.41, 9.97]，全部<10.0
- PAE>10的序列在AF3阶段通过率为0%

**预期收益**：
- 剔除PAE>10的低质量序列，预计减少数百条"虚假通过"
- RF2通过序列平均PAE显著下降
- 与v2.0原始PAE阈值一致，回归经过验证的标准

**潜在风险**：
- 中等风险：可能排除少量PAE在10-12之间但结构仍可挽救的序列
- 但数据显示：v2.0中PAE>10的序列FC也为0，因此实际风险很低

**性能影响**：
- RF2通过数预计从2878→约2300-2500
- 下游计算量减少约20-30%
- 整体管线运行时间缩短

### 3.3 移除首残基N

**数据支撑**：
- v2.0中N开头：RF2通过109条，FC=0（0.00%）
- v2.1中N开头：RF2通过462条，FC=0（0.00%）
- 无论哪个版本，N开头序列从未成为最终候选

**预期收益**：
- 减少约462条无效序列进入下游
- 生成器不再产生N开头的CDR3序列
- 策略更精准，减少计算浪费

**潜在风险**：
- 风险极低：N开头FC始终为0，无有效候选损失
- 理论上可能存在N开头+特定组合能通过的场景，但数据中未观察到

**性能影响**：
- 首残基白名单从15个减至14个
- 生成器搜索空间略微缩小
- 下游计算量减少约16%（462/2908）

### 3.4 RMSD阈值维持2.5

**数据支撑**：
- v2.0中rmsd≤2.0与v2.1中rmsd≤2.5的差异序列数较少
- RMSD对最终候选的影响小于lddt和PAE

**预期收益**：
- 保留v2.1的放宽效果，允许更多结构变异
- 与lddt/PAE收紧形成平衡

**潜在风险**：
- 低风险：RMSD放宽2.0→2.5引入的序列质量可接受

## 六、回滚方案

如果v2.2效果不理想，可通过修改csco_config.py中的DEFAULT_CONFIG快速回滚：

```python
# 回滚到v2.1
"rf2_lddt_threshold": 0.85,
"rf2_pae_threshold": 12.0,

# 回滚到v2.0
"rf2_lddt_threshold": 0.88,
"rf2_pae_threshold": 10.0,
"rf2_rmsd_threshold": 2.0,
```
