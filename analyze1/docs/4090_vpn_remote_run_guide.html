CSC-O：VPN + 4090 远程托管运行指南

目标：输入你的 10000 条 CSV 数据源，在导师提供的 4090 服务器上（需先连 openconnect VPN）后台执行，并支持随时查看进度与输出。

仓库入口csco_pipeline.py（支持 --input/--device auto/--resume）

1）你需要准备什么

  • 本地已安装并配置好 openconnect（校内 VPN）。
  • 远程 4090 服务器：导师提供的 IP、账号、密码（建议后续改为 SSH key）。
  • 数据：你的 CSV（10000 行）。

CSV 最低要求（强烈建议）

管线会自动做列名别名映射与补齐缺失列，但为了避免“有效数据不足”报错，建议确保至少这些列存在且不全为空： vh_sequence（或 full_sequence / sequence / seq）、 rf2_pred_lddt（或 plddt）、 rf2_interaction_pae（或 pae）。

注意：如果 rf2_pred_lddt / rf2_interaction_pae 两列几乎都为空，会触发：有效数据不足，需要至少50条有RF2指标的记录。

2）本地开启 VPN（openconnect）

你说本地已配置好校内 VPN，这里只给出“后台运行”的常用写法（按你的实际配置替换）。

# 示例：后台开启 VPN（按你现有校内 VPN 配置替换）
# 方式A：nohup（适合简单场景）
nohup sudo openconnect <你的VPN地址或配置> > vpn.log 2>&1 &

# 方式B：tmux（推荐，掉线后可重新 attach）
tmux new -s vpn
sudo openconnect <你的VPN地址或配置>
# 按 Ctrl+b 再按 d 退出到后台

验证 VPN：连上后再执行后面的 ssh，能通就说明链路 OK。

3）连接 4090 服务器（SSH）

把下面的占位符替换成导师给你的信息：

# 推荐：第一次连接时确认指纹（不要无脑 yes）
ssh <USER>@<IP>

# 如果必须用密码（不推荐长期这样），会提示你输入密码
# 建议后续改成 SSH Key + 禁用密码登录（由管理员/导师决定）

在服务器上做一次 GPU/环境检查

# 4090 是否可用
nvidia-smi

# python 是否存在
python3 --version || python --version

4）把 CSV 上传到服务器

建议在服务器上建一个固定工作目录（例如 ~/csco_runs）。

# 在服务器上
mkdir -p ~/csco_runs/data ~/csco_runs/out ~/csco_runs/work

然后从本地把 CSV 传上去（任选一种）：

# 本地执行（scp）
scp "/path/to/your_10000.csv" <USER>@<IP>:~/csco_runs/data/input.csv

# 或者 rsync（断点续传更稳）
rsync -avP "/path/to/your_10000.csv" <USER>@<IP>:~/csco_runs/data/input.csv

5）把代码同步到服务器

有两种常用方式：

  • 方式A（推荐）：服务器上 git clone 你的仓库（需要仓库可访问）。
  • 方式B：本地用 rsync 把整个代码目录同步过去（适合私有/无 git 访问）。
# 方式B：从本地同步当前仓库到服务器（排除 output/work 等大文件目录）
rsync -avP --exclude "output/" --exclude "work/" --exclude "__pycache__/" \
  "./analyze1/" \
  <USER>@<IP>:~/csco_runs/repo/

6）在服务器上安装依赖（一次性）

本仓库在 ESM-2、lifelines、lightgbm、sklearn、matplotlib/seaborn 等处会用到第三方库。最稳妥做法是用虚拟环境。

# 在服务器上
cd ~/csco_runs/repo

python3 -m venv .venv
source .venv/bin/activate

# 基础依赖（如服务器已预装可跳过对应项）
python -m pip install -U pip wheel setuptools
python -m pip install pandas numpy matplotlib seaborn scikit-learn scipy lifelines lightgbm

# ESM-2 相关（可能需要 torch + esm）
# torch 安装方式依赖 CUDA/驱动环境，若服务器已装好 torch+cuda 可先跳过
python -m pip install fair-esm

注意：torch 的 CUDA 版安装命令与服务器 CUDA/驱动版本强相关。如果这里卡住，最有效的排查顺序是：nvidia-smi → python -c "import torch; print(torch.cuda.is_available())"。

7）后台运行（托管在云端）

推荐用 tmux 托管训练/推理进程：断开 SSH 不会中断；随时能 attach 回去看日志。

# 在服务器上
cd ~/csco_runs/repo
source .venv/bin/activate

tmux new -s csco

# 在 tmux 里执行（用 4090：--device cuda；断点续跑：--resume）
python csco_pipeline.py \
  --input ~/csco_runs/data/input.csv \
  --output ~/csco_runs/out \
  --work ~/csco_runs/work \
  --device cuda \
  --batch-size 4 \
  --top-n 2000 \
  --resume \
  > ~/csco_runs/out/run.log 2>&1

# 按 Ctrl+b 再按 d：把 tmux 放到后台

随时回来看：

# 重新连接
tmux attach -t csco

# 查看 tmux 会话列表
tmux ls

8）随时查看：用网页浏览输出目录（推荐：SSH 端口转发）

最简单的“云端可随时查看”方式：在服务器上用 python -m http.server 把输出目录起一个只在服务器本地监听的静态文件服务， 然后通过 SSH 端口转发在本地浏览器打开。 

# 服务器上：起一个只监听 127.0.0.1 的静态文件服务（更安全）
cd ~/csco_runs/out
tmux new -s csco-http
python -m http.server 8000 --bind 127.0.0.1
# Ctrl+b d 退出到后台

# 本地：做 SSH 端口转发（保持该终端窗口不关）
ssh -L 8000:127.0.0.1:8000 <USER>@<IP>

# 然后本地浏览器打开：
# http://127.0.0.1:8000/
# 你可以直接点开 run.log、以及输出的 png/csv/json/txt

安全提醒：不要把 http.server 直接绑定到 0.0.0.0 并暴露公网端口，除非你非常确认服务器防火墙/访问控制已正确配置。

9）常见问题（快速排查）

（A）提示有效数据不足

# 检查 CSV 列名（在服务器上）
python - <<'PY'
import pandas as pd
df = pd.read_csv("/home/<USER>/csco_runs/data/input.csv")
print(df.columns.tolist()[:50])
print("n_rows:", len(df))
print("non_na rf2_pred_lddt:", df.get("rf2_pred_lddt", pd.Series([])).notna().sum())
print("non_na rf2_interaction_pae:", df.get("rf2_interaction_pae", pd.Series([])).notna().sum())
PY

如果你的列叫 plddt/pae 也可以；入口脚本会做别名映射。

（B）ESM-2 很慢或 OOM

  • 把 --batch-size 调小（例如 1 或 2）。
  • 确认在 4090 上跑：--device cuda，且 nvidia-smi 能看到进程占用。

（C）断线后如何继续

  • 只要你用的是 tmux，断线不影响运行；重新 SSH 后 tmux attach -t csco 即可。
  • 管线支持断点续跑：加 --resume，会自动跳过已完成阶段（状态在 work/pipeline_state.json）。

10）你最终会得到哪些输出

  • 核心表：feature_matrix.csv、survival_data.csv
  • ESM-2：esm2_embeddings.npy（非常大）
  • 反事实建议：counterfactual_suggestions.csv、truncation_suggestions.csv
  • 策略汇总：design_strategy.json、design_strategy.txt、csco_analysis_report.txt
  • 图：*.png（漏斗、森林图、t-SNE、热图等）
