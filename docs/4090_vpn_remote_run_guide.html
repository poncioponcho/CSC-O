1 数据准备：10000条CSV输入

1.1 输入文件要求

参数说明示例
格式CSV (逗号分隔) 或 XLSX.csv / .xlsx
行数建议 10000 条以上10572 条已测试通过
编码UTF-8避免中文列名或特殊字符
大小通常 < 200MB含完整序列文本

1.2 必需列清单

列名类型必需性别名支持默认值(缺失时)
vh_sequencestring必需vh_seq, heavy_chain, vh, full_sequence''(空)
rf2_pred_lddtfloat必需pred_lddt, plddtNaN
rf2_interaction_paefloat必需interaction_pae, paeNaN
rf2_passed_filterbool/int推荐passed_filter, rf2_passFalse
final_candidatebool/int推荐is_candidate, selectedFalse
cdr3_sequencestring推荐cdr3, h3_sequence从VH序列自动提取
funnel_stagestring推荐stage自动推断
af3_analyzedbool可选-False
schrodinger_analyzedbool可选-False
backbone_idint可选backbone0

1.3 数据校验命令（本地先跑）

$ python3 -c "
import pandas as pd
df = pd.read_csv('你的数据.csv')
print(f'行数: {len(df)}')
print(f'vh_sequence 非空: {df[\"vh_sequence\"].notna().sum()}')
print(f'rf2_pred_lddt 有效: {df[\"rf2_pred_lddt\"].notna().sum()}')
print(f'rf2_interaction_pae 有效: {df[\"rf2_interaction_pae\"].notna().sum()}')
"
⚠️ 如果 rf2_pred_lddt 或 rf2_interaction_pae 全为空，管线会在 DataAdapter.validate() 阶段报错：
ValueError: 有效数据不足 (n 行), 需要至少50条有RF2指标的记录
请确保这两列有实际数值。

1.4 管线自动补全能力

代码中的 DataAdapter 模块会自动处理以下情况：

  • 缺失列 → 按81列完整模式自动填充默认值（NaN/False/''）
  • 列名别名 → 自动映射到标准名称（如 plddt → rf2_pred_lddt）
  • 布尔值格式 → 支持 True/False/1/0/yes/no/1.0/0.0 多种变体
  • VH序列缺失 → 从 full_sequence 回退
  • CDR3缺失 → 从VH序列启发式提取（WGQGTLVTVS motif）
  • funnel_stage缺失 → 从各阶段标志自动推断

2 VPN连接：OpenConnect 配置

2.1 前置条件确认

检查项状态操作
OpenConnect 已安装需确认which openconnect
VPN配置文件 (.ovpn/.conf)需确认通常在 /etc/openconnect/ 或导师提供
服务器IP地址待填写见下方配置区
账号密码待填写见下方配置区

2.2 连接命令（本地终端）

# 方式A：前台运行（调试用）
$ sudo openconnect <VPN_SERVER_IP> \
  --user=<USERNAME> \
  --authgroup=<GROUP_NAME> \
  --protocol=anyconnect

# 方式B：后台运行（推荐生产使用）
$ sudo openconnect <VPN_SERVER_IP> \
  --user=<USERNAME> \
  --background \
  --pid-file=/tmp/vpn.pid \
  --script=/etc/vpnc-script-default \
  --passwd-on-stdin <<< <PASSWORD>

# 方式C：使用配置文件（如果导师提供了.ovpn）
$ sudo openconnect --config=/path/to/school.vpn.conf \
  --background --pid-file=/tmp/vpn.pid

2.3 验证VPN连通性

$ ping <GPU_SERVER_INTERNAL_IP>
# 应该能ping通内网服务器

$ ssh <USERNAME>@<GPU_SERVER_INTERNAL_IP>
# 应该能成功SSH登录
❌ 常见问题：
• openconnect: command not found → 安装：brew install openconnect (macOS) 或 sudo apt install openconnect (Linux)
• SSL connect error → 检查VPN证书是否过期，或加 --no-cert-check（不安全但可用）
• Authentication failed → 确认用户名/密码/认证组正确

3 SSH 登录与文件传输

3.1 SSH 登录 GPU 服务器

$ ssh <USERNAME>@<GPU_SERVER_INTERNAL_IP> -p <SSH_PORT>

# 首次登录会提示保存指纹，输入 yes 即可
# The authenticity of host 'xxx' can't be established.
# ECDSA key fingerprint is SHA256:xxxxxx.
# Are you sure you want to continue connecting (yes/no/[fingerprint])? yes

3.2 上传数据文件（本地→服务器）

# 上传CSV数据文件到服务器
scp -P <SSH_PORT> \
  ~/Desktop/你的数据.csv \
  <USERNAME>@<GPU_SERVER_INTERNAL_IP>:~/csco_runs/data/input.csv

# 上传整个CSC-O代码目录
$ scp -rP -P <SSH_PORT> \
  ~/Desktop/课题组/蛋白质结构序列实验_结构方向优化/analyze1/ \
  <USERNAME>@<GPU_SERVER_INTERNAL_IP>:~/csco_runs/code/

3.3 在服务器上创建工作目录

# 登录后执行
server$ mkdir -p ~/csco_runs/{data,code,out,work}
server$ ls -la ~/csco_runs/
# drwxr-xr-x  data/ code/ out/ work/
✅ 目录结构说明：
• data/ — 放置输入CSV文件
• code/ — 放置CSC-O源码（所有.py文件）
• out/ — 输出结果（分析报告、可视化图、策略JSON等）
• work/ — 工作缓存（pipeline_state.json断点续跑）

4 环境部署：依赖安装

4.1 检查系统环境

server$ python3 --version          # 需要 >= 3.10
server$ nvidia-smi                  # 确认4090可见
server$ nvcc --version             # CUDA版本
server$ free -h                     # 内存 >= 16GB
server$ df -h | head -5           // 磁盘空间 >= 10GB

4.2 创建虚拟环境并安装依赖

server$ cd ~/csco_runs
server$ python3 -m venv venv
server$ source venv/bin/activate

# 核心依赖（一次性安装全部）
(venv)$ pip install --upgrade pip
(venv)$ pip install pandas numpy scipy scikit-learn matplotlib seaborn openpyxl statsmodels fairlearn
(venv)$ pip install lightgbm torch torchvision
(venv)$ pip install fair-esm          # ESM-2蛋白质语言模型

# 验证关键包
(venv)$ python3 -c "import esm; print('ESM OK:', esm.__version__)"
(venv)$ python3 -c "import torch; print('CUDA:', torch.cuda.is_available(), torch.cuda.get_device_name(0))"
(venv)$ python3 -c "import lifelines; import lightgbm; print('All deps OK')"
⚠️ 关于 PyTorch 版本：如果服务器CUDA版本较旧（如11.x），可能需要指定PyTorch版本：
pip install torch==2.0.1+cu118 torchvision==0.15.2+cu118 -f https://download.pytorch.org/whl/cu118/torch_stable.html
先用 nvidia-smi 查看CUDA版本，再选择对应PyTorch。

4.3 一键安装脚本（可复制到服务器直接运行）

5 运行管线：后台托管执行

5.1 完整运行命令

# 激活环境
server$ source ~/csco_runs/venv/bin/activate
server$ cd ~/csco_runs/code

# ★★★ 主运行命令 ★★★
(venv)$ KMP_DUPLICATE_LIB_OK=TRUE python3 csco_pipeline.py \
  --input ~/csco_runs/data/input.csv \
  --output ~/csco_runs/out \
  --work ~/csco_runs/work \
  --device cuda \
  --batch-size 4 \
  --top-n 8000 \
  --resume \
  > ~/csco_runs/out/run.log 2>&1 &

# 记录进程PID
server$ echo $! > ~/csco_runs/out/pid.txt
server$ cat ~/csco_runs/out/pid.txt   # 显示PID号

5.2 使用 screen/tmux 保持会话（强烈推荐）

# 创建一个持久化终端会话
server$ screen -S csco

# 在screen内部激活环境并运行
server# source ~/csco_runs/venv/bin/activate
server# cd ~/csco_runs/code
server# KMP_DUPLICATE_LIB_OK=TRUE python3 csco_pipeline.py \
  --input ~/csco_runs/data/input.csv \

# 按 Ctrl+A 再按 D 脱离会话（程序继续运行）
# 重新连接: screen -r csco

5.3 参数详解

参数缩写默认值说明
--input-i必填输入CSV/XLSX路径
--output-o./output输出目录
--work-w./work工作/缓存目录
--device-dauto计算设备: cuda/mps/cpu
--batch-size-b4ESM-2批大小（4090可用8-16）
--top-n-n2000反事实编辑处理的失败序列数
--resume-r关闭断点续跑（跳过已完成阶段）

5.4 6个阶段的预期输出

阶段名称耗时估计关键输出文件
1data_engineering~30秒feature_matrix.csv, survival_data.csv
2layer1_stratified~2分钟cox_hazard_ratios.csv, *.png ×5
3layer2_causal~3分钟ate_estimates.csv, *.png ×3
4layer3_esm2_encode~4-6小时esm2_embeddings.npy (20MB)
5layer3_counterfactual~30分钟counterfactual_suggestions.csv, *.png ×3
6layer5_synthesis~1分钟design_strategy.json, design_strategy.txt, csco_analysis_report.txt
✅ 断点续跑机制：每个阶段完成后状态写入 work/pipeline_state.json。
如果中途中断（如网络断开），重新执行相同命令加 --resume 即可跳过已完成阶段继续。
ESM-2编码完成后生成的 esm2_embeddings.npy 会被缓存，不会重新编码。

8 常见问题排查

Q1: VPN连接后无法SSH到GPU服务器

# 检查路由
$ traceroute <INTERNAL_IP>
# 如果不通，尝试指定VPN子网的路由
$ ip route add <SUBNET> via <VPN_GATEWAY>

Q2: ESM-2模型下载很慢或超时

首次运行会从Facebook服务器下载模型权重（~300MB）。如果服务器在国内网络受限：

# 设置镜像代理
(venv)$ export HF_ENDPOINT=https://hf-mirror.com
# 或手动下载后放到cache目录
(venv)$ mkdir -p ~/.cache/torch/hub/checkpoints
(venv)$ wget -O ~/.cache/torch/hub/checkpoints/esm2_t12_35M_UR50D.pt https://dl.fbaipublicfiles.com/fair-esm/models/esm2_t12_35M_UR50D.pt

Q3: OMP错误（libomp）

# macOS/Linux常见问题
(venv)$ KMP_DUPLICATE_LIB_OK=TRUE python3 csco_pipeline.py ...
# 或者设置环境变量
(venv)$ export KMP_DUPLICATE_LIB_OK=TRUE

Q4: 显存不足（OOM）

# 减小batch_size
(venv)$ python3 csco_pipeline.py ... --batch-size 1

# 或减少top-n（反事实分析的序列数）
(venv)$ python3 csco_pipeline.py ... --top-n 1000

Q5: 中途断开后如何恢复

# 直接重新运行相同命令（加上 --resume）
(venv)$ KMP_DUPLICATE_LIB_OK=TRUE python3 csco_pipeline.py \
  --input ~/csco_runs/data/input.csv \
  --output ~/csco_runs/out \
  --work ~/csco_runs/work \
  --device cuda --batch-size 4 --top-n 8000 --resume \
  > ~/csco_runs/out/run.log 2>&1 &

# 查看当前进度
(venv)$ cat ~/csco_runs/work/pipeline_state.json | python3 -m json.tool

Q6: 如何确认运行是否完全成功

# 最终成功的标志
server$ grep "管线完成" ~/csco_runs/out/run.log
# 应输出: 管线完成！输出目录: /home/user/csco_runs/out

# 检查所有6个阶段都完成了
server$ grep "阶段完成" ~/csco_runs/out/run.log
# 应输出6行（对应6个阶段）

# 检查最终产物是否存在
server$ ls -lh ~/csco_runs/out/{design_strategy.json,design_strategy.txt,csco_analysis_report.txt,csco_dashboard.png}
